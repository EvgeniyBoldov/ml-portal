# ML Portal Roadmap

## 2026: Platform Maturity Plan

Roadmap синхронизирован с `CONCEPT.md`: фокус на надежности платформы, управляемости агентов и enterprise-ready эксплуатации.

## Phase 1 — Stabilize Core (ближайшие 1–2 спринта)

### Цели

1. Устранить архитектурные риски в backend/core flows.
2. Зафиксировать единые правила фронта/бэка и убрать расхождения.
3. Закрыть критичный техдолг, влияющий на надежность релизов.

### Инициативы

- [ ] Миграции: устранить конфликты revision id и стабилизировать migration chain.
- [ ] Транзакции: выровнять `flush/commit` policy во всех API/репозиториях.
- [ ] Рефактор крупных сервисов без изменения API-контрактов.
- [ ] Убрать debug/console шум из production paths.
- [ ] Довести документацию правил/паттернов до единого стандарта.

### Критерии выхода из фазы

- Нет конфликтующих миграций.
- Нет нарушений transaction policy в критичных эндпойнтах.
- Unit/integration тесты покрывают ключевые бизнес-правила (permissions/credentials/routing).

## Phase 2 — Operational Excellence (Q2–Q3 2026)

### Цели

1. Сделать платформу наблюдаемой и управляемой в ежедневной эксплуатации.
2. Повысить предсказуемость поведения агентов и tool integrations.

### Инициативы

- [ ] Dashboard по latency/error/success-rate/tool-health.
- [ ] Улучшение health-check и нотификаций по инцидентам.
- [ ] Расширенный audit trail (API + UI).
- [ ] Оптимизация производительности backend и admin UI.
- [ ] Политики лимитов/квот по tenant (rate/cost controls).

### Критерии выхода из фазы

- Мониторинг покрывает все critical flows.
- Есть эксплуатационные runbooks для типовых инцидентов.
- p95 и error-rate укладываются в согласованные SLO.

## Phase 3 — Product Scale (Q3–Q4 2026)

### Цели

1. Масштабировать продуктовые сценарии без деградации управляемости.
2. Ускорить запуск новых агентных кейсов для бизнес-команд.

### Инициативы

- [ ] Улучшение lifecycle версий агентов/промптов/политик.
- [ ] RBAC UX на уровне интерфейса (скрытие/ограничение действий).
- [ ] Каталог шаблонов (agents/prompts/policies) с контролируемым rollout.
- [ ] Cost/usage аналитика по tenant и сценариям.

### Критерии выхода из фазы

- Time-to-configure нового агента снижен.
- Рост доли успешных agent runs без роста инцидентов.

## Phase 4 — Enterprise Readiness (конец 2026)

### Цели

1. Закрепить enterprise-профиль по безопасности и надежности.
2. Подготовить платформу к масштабированию инфраструктуры.

### Инициативы

- [ ] Security hardening (ключи, доступы, аудит, секреты).
- [ ] Расширение тестового контура (critical E2E + нагрузочные сценарии).
- [ ] Проработка multi-region стратегии и operational playbooks.

## KPI и контроль прогресса

Отслеживаем ежемесячно:

- Agent success rate
- p95 latency (chat/rag/tool)
- Error rate (по типам)
- Tool availability
- MTTR по critical incidents
- Time-to-configure нового агента

## Definition of Done (для инициатив roadmap)

Задача считается завершенной, если:

1. Код прошел review и соответствует RULES/PATTERNS.
2. Есть релевантные тесты (unit/integration; для критичных — e2e).
3. Обновлена документация и операционные заметки.
4. Изменение проверено в staging.
