# Видение продукта

## Что это

**ML Portal** — корпоративный AI-ассистент для департаментов компании.

## Цели

1. **Единая точка входа** — один интерфейс для всех AI-задач: чат, RAG, анализ документов
2. **Мультитенантность** — каждый департамент работает изолированно, но может получить доступ к ресурсам других через политики
3. **Гибкие агенты** — конфигурируемые AI-агенты с инструментами, промптами и политиками
4. **Контроль доступа** — гранулярные права на уровне User > Tenant > Default
5. **Оффлайн-работа** — все модели и данные хранятся локально, без внешних зависимостей в проде

## Ключевые сценарии

### Чат с агентом
Пользователь общается с AI-агентом, который использует инструменты (RAG, Jira, NetBox) для решения задач.

### RAG Pipeline
Загрузка документов → извлечение текста → чанкинг → эмбеддинг → векторный поиск.

### Коллекции
Структурированные данные с SQL и векторным поиском для специализированных задач.

### Администрирование
Управление пользователями, агентами, промптами, инструментами, правами доступа.

## Не цели (out of scope)

- Публичный SaaS — это внутренний продукт
- Real-time collaboration — один пользователь = один чат
- Мобильное приложение — только веб-интерфейс
- Multi-region — один кластер на организацию

## Целевая аудитория

- **Пользователи**: сотрудники департаментов, работающие с AI-ассистентом
- **Tenant Admin**: администраторы департаментов, настраивающие агентов и права
- **Super Admin**: администраторы платформы, управляющие всей системой

## Метрики успеха

- Время ответа агента < 5 секунд (p95)
- Доступность системы > 99%
- Качество RAG-поиска (relevance) > 80%
- Adoption rate > 50% сотрудников департамента
