# ML Portal — Backend Architecture

> **Дата**: 2026-04-06
> **Статус**: Production-ready (v8.0/10)
> **Scope**: 529 .py файлов, 82K LOC, 80 тестов

## 0. Обзор

**Backend** — асинхронный FastAPI + SQLAlchemy, разделённый на слои:
- **Models** (36 файлов) — SQLAlchemy ORM с UUID PK, `created_at`, `updated_at`
- **Repositories** (27 файлов) — Data access layer с CRUD, pagination, tenant isolation
- **Services** (70+ файлов) — Business logic, async-first, domain exceptions
- **Routers** (18+ sub-routers) — REST API v1, модульная структура по доменам
- **Agents** (runtime engine) — Pipeline → Planner → AgentToolRuntime, 2-level execution
- **Core** (infrastructure) — DB, auth, middleware, DI, exceptions, logging

**Ключевые фичи**:
- Двухуровневая агентская архитектура (стратегия + тактика)
- Sandbox с git-like branches и immutable snapshots для экспериментов
- Единый каталог domain exceptions с HTTP mapping
- RBAC v3 flat rules (user > tenant > platform priority)
- Versioned entities (Agent/AgentVersion, Tool/ToolRelease, Collection/CollectionVersion)
- SSE runtime events для наблюдаемости

## 1. Архитектурные слои

### 1.1 Models (`src/app/models/`)

36 моделей SQLAlchemy с полной типизацией:

```
Agent ──┬─→ AgentVersion (versioned)
        ├─→ AgentBinding (tool pinning)
        └─→ AgentRun (execution trace)

Tool ──┬─→ ToolRelease (versioned)
       ├─→ ToolInstance (concrete connection)
       └─→ DiscoveredTool (raw catalog)

Collection ──┬─→ CollectionVersion (versioned)
             ├─→ CollectionField (schema)
             └─→ CollectionRow (data)

Tenant ──┬─→ User (scoped)
         ├─→ PermissionSet (RBAC)
         └─→ CredentialSet (encrypted)

Sandbox ──┬─→ SandboxBranch (git-like)
          ├─→ SandboxBranchOverride (state)
          └─→ SandboxRun (execution)
```

**Обязательные поля**:
```python
id: UUID (primary key)
created_at: datetime (UTC)
updated_at: datetime (UTC, auto-update)
```

**Versioning pattern**:
- Container (Agent, Tool, Collection) — human metadata, slug, is_active
- Version (AgentVersion, ToolRelease, CollectionVersion) — config, prompt, schema

### 1.2 Repositories (`src/app/repositories/`)

Generic `AsyncRepository[T]` base с:
- CRUD operations (create, read, update, delete)
- Pagination (offset/limit + cursor-based)
- Tenant isolation (scope filtering)
- Error mapping (DB errors → domain exceptions)
- Optimistic locking support

**Пример**:
```python
class AgentRepository(AsyncRepository[Agent]):
    async def get_by_slug(self, slug: str) -> Agent | None:
        # Tenant-scoped query
        stmt = select(Agent).where(Agent.slug == slug)
        return await self.session.scalar(stmt)
```

### 1.3 Services (`src/app/services/`)

**Правила**:
- Только async методы
- Полная типизация
- Кидают domain exceptions, не HTTPException
- Используют repositories для DB access
- Вызывают `await session.flush()` в repo, `await session.commit()` в service

**Крупные сервисы**:
- `collection_service.py` (1314 строк) — CRUD + DDL + vector + search + CSV (частично декомпозирован)
- `rag_status_manager.py` (874 строки) — статус-машина для документов
- `system_llm_executor.py` (805 строк) — triage LLM (3 role types)
- `tool_instance_service.py` (693 строки) — instance resolution + credentials
- `agent_service.py` (549 строк) — agent CRUD + versioning

**Структура**: 1 домен = 1 файл или пакет


## 2. Agent Runtime (Глубокий анализ)

### 2.1 Pipeline (8 этапов)

```
RuntimePipeline.execute()
├── 1. Resolve agent slug (tenant default или fallback)
├── 2. Load platform config (limits, policies)
├── 3. Triage (SystemLLMExecutor → final/clarify/orchestrate)
├── 4. Early exits (final answer / clarify prompt)
├── 5. ExecutionPreflight
│   ├── AgentResolver (load Agent + AgentVersion + AvailableActions)
│   └── ToolResolver (resolve instances, credentials, operations)
├── 6. Availability check (все ли инструменты доступны?)
├── 7. Build outline (HelperSummary + ExecutionOutline)
└── 8. PlannerRuntime.execute()
    └── Loop: Planner LLM → AGENT_CALL → AgentToolRuntime → facts → synthesis
        └── AgentToolRuntime.execute()
            └── Loop: LLM → operation_call → execute → results → LLM
```

### 2.2 Компоненты

| Компонент | Файл | Строк | Роль |
|---|---|---|---|
| **ExecutionPreflight** | `execution_preflight.py` | 408 | Facade над resolvers |
| **AgentResolver** | `agent_resolver.py` | 101 | Load Agent + Version |
| **ToolResolver** | `tool_resolver.py` | 437 | Resolve tools + semantics |
| **OperationRouter** | `operation_router.py` | 690 | Find operations, build ResolvedOperation |
| **PromptAssembler** | `prompt_assembler.py` | 290 | Modular prompt construction |
| **PolicyEngine** | `policy_engine.py` | 190 | Runtime policy enforcement |
| **PlannerRuntime** | `runtime/planner.py` | 691 | Planner loop + synthesis |
| **AgentToolRuntime** | `runtime/agent.py` | 357 | Agent tool-call loop |
| **RuntimePipeline** | `runtime/pipeline.py` | 491 | Unified orchestrator |
| **SubAgentDispatcher** | `runtime/sub_agent_dispatcher.py` | 261 | Handle AGENT_CALL actions |

### 2.3 Contracts (DTOs)

`agents/contracts.py` — 280 строк чистых Pydantic/dataclass DTOs:
- `ResolvedDataInstance`, `ResolvedOperation`, `ProviderExecutionTarget` — runtime DTOs
- `OperationAction`, `AgentAction`, `AvailableActions` — planner whitelist
- `NextAction`, `Observation`, `PolicyDecision` — planner protocol

## 4. Data Layer

### 4.1 Tenant Isolation

Все запросы фильтруются по `tenant_id`:

```python
async def get_collection(self, collection_id: UUID, tenant_id: UUID) -> Collection:
    collection = await self.repo.get_by_id(collection_id)
    if not collection or collection.tenant_id != tenant_id:
        raise CollectionNotFoundError(collection_id)
    return collection
```

### 4.2 Credential Encryption

Credentials зашифрованы через `CryptoService` (Fernet):
```python
encrypted = crypto.encrypt({"api_key": "...", "token": "..."})
decrypted = crypto.decrypt(encrypted)
```

### 4.3 RBAC v3

Flat resource-level rules с приоритетом: **User > Tenant > Platform**

```python
# Resolve effective permission
user_perm = await rbac.get_by_scope("user", user_id=user_id)
tenant_perm = await rbac.get_by_scope("tenant", tenant_id=tenant_id)
platform_perm = await rbac.get_by_scope("platform")

# User override allowed even if platform denies
effective = user_perm or tenant_perm or platform_perm
```

```python

    http_status: int = 500
    code: str = "INTERNAL_ERROR"

class NotFoundError(AppError):

    code = "NOT_FOUND"

class ReleasePinnedError(ProtectedError): ...
```

### 5.2 Exception Handlers

Автоматический HTTP mapping через `install_exception_handlers`:

```python
async def app_error_handler(request: Request, exc: AppError) -> JSONResponse:
    return JSONResponse(
        status_code=exc.http_status,
        content={"code": exc.code, "detail": str(exc), "trace_id": request_id}
    )
```

**Backward compat**: `core/errors.py` — 18-строчный shim для legacy imports.

## 6. Security & Middleware

### 6.1 Middleware Stack

```python
app.add_middleware(TimeoutMiddleware)      # 120s default, SSE excluded
app.add_middleware(TenantMiddleware)       # JWT fallback + UUID validation
app.add_middleware(RequestContextMiddleware)  # Correlation ID (X-Request-Id)
app.add_middleware(CORSMiddleware)         # CORS config
```

### 6.2 Auth

- JWT с RSA/HS256
- Argon2 + pepper для паролей
- `create_access_token` с `jti`, `iss`, `aud`
- Refresh tokens в httpOnly cookies
- `UserCtx` dataclass в DI chain

### 6.3 Rate Limiting

Redis-based rate limiting:
- `POST /auth/login` — 5 попыток/минуту
- `POST /chats/{chat_id}/messages` — 30 запросов/минуту
