# План восстановления после сбоя (Disaster Recovery Plan)
**Компания:** Дубалайн  
**Версия:** 1.0  
**Дата утверждения:** 20.02.2026  

---

## 1. Общие положения

### 1.1. Цель документа
Настоящий документ определяет процедуры и стратегии восстановления критически важной сетевой инфраструктуры компании Дубалайн после различных типов сбоев и катастроф.

### 1.2. Область применения
План распространяется на всю сетевую инфраструктуру дата-центров Дубалайн, включая:
- Core и Distribution уровни сети
- Сетевое оборудование (Cisco, Juniper, Arista)
- Системы управления и мониторинга
- Системы питания и охлаждения

### 1.3. Критичные системы
- **Уровень критичности 1:** Core switches, маршрутизаторы, системы управления
- **Уровень критичности 2:** Distribution switches, системы хранения
- **Уровень критичности 3:** Access switches, системы мониторинга

---

## 2. Словарь терминов

| Термин | Определение |
|--------|------------|
| **RTO** (Recovery Time Objective) | Максимально допустимое время восстановления системы |
| **RPO** (Recovery Point Objective) | Максимально допустимая потеря данных |
| **DR Site** | Резервная площадка для восстановления |
| **Failover** | Переключение на резервные системы |
| **Hot Standby** | Горячий резерв (готов к работе) |
| **Cold Standby** | Холодный резерв (требует настройки) |
| **BCP** (Business Continuity Plan) | План непрерывности бизнеса |
| **HA** (High Availability) | Высокая доступность |

## 3. Классификация сбоев

### 3.1. Уровни катастроф

#### Уровень 1: Локальный сбой
- Отказ одного устройства
- Отказ одного модуля
- Программный сбой

#### Уровень 2: Зональный сбой
- Отказ стойки оборудования
- Отказ системы питания в стойке
- Отказ системы охлаждения

#### Уровень 3: Полный сбой дата-центра
- Потеря электропитания всего ЦОД
- Пожар или затопление
- Сетевой сегментация

#### Уровень 4: Географическая катастрофа
- Землетрясение
- Террористический акт
- Глобальный сбой инфраструктуры

### 3.2. Матрица RTO/RPO

| Уровень критичности | RTO | RPO | Стратегия восстановления |
|---------------------|-----|-----|-------------------------|
| Уровень 1 | 15 минут | 5 минут | Hot Standby |
| Уровень 2 | 1 час | 30 минут | Warm Standby |
| Уровень 3 | 4 часа | 2 часа | Cold Standby |

## 5. Процедуры восстановления

### 5.1. План действий при сбое

#### Шаг 1: Идентификация сбоя (0-5 минут)
```bash
# Проверка состояния оборудования
show system status
show environment
show interfaces status
show logging
```

#### Шаг 2: Оценка масштаба (5-15 минут)
- Определить уровень катастрофы
- Проверить доступность резервных систем
- Оценить время восстановления

#### Шаг 3: Активация DR плана (15-30 минут)
- Уведомить команду реагирования
- Запустить процедуры failover
- Начать восстановление критичных сервисов

#### Шаг 4: Восстановление сервисов (30 минут - 4 часа)
- Восстановить сетевую связность
- Проверить работоспособность сервисов
- Провести валидацию

#### Шаг 5: Возврат к нормальной работе (4-24 часа)
- Планирование возврата на основную площадку
- Тестирование систем
- Пост-инцидентный анализ

### 5.2. Процедура Failover

#### Автоматический Failover (для критичных систем)
```python
# Скрипт автоматического переключения
def auto_failover():
    if check_primary_health() == False:
        if check_dr_health() == True:
            activate_dr_site()
            update_dns_records()
            notify_team()
            return True
    return False
```

#### Ручной Failover
```bash
# 1. Проверка состояния DR площадки
ssh dr-core-01 "show system status"

# 2. Активация DR конфигурации
ssh dr-core-01 "configure terminal"
ssh dr-core-01 "interface Ethernet1/1"
ssh dr-core-01 "no shutdown"
ssh dr-core-01 "exit"
ssh dr-core-01 "write memory"

# 3. Обновление маршрутизации
ssh dr-core-01 "router ospf 1"
ssh dr-core-01 "redistribute static"
ssh dr-core-01 "exit"
```

### 6.3. Скрипт мониторинга здоровья

```python
#!/usr/bin/env python3
import subprocess
import smtplib
from email.mime.text import MIMEText

def check_device_health(ip):
    try:
        result = subprocess.run(
            ['ping', '-c', '3', ip],
            capture_output=True,
            text=True,
            timeout=10
        )
        return result.returncode == 0
    except:
        return False

def send_alert(message):
    msg = MIMEText(message)
    msg['Subject'] = 'DR Alert - System Failure'
    msg['From'] = 'dr-monitor@dubalaine.ru'
    msg['To'] = 'network-team@dubalaine.ru'
    
    with smtplib.SMTP('localhost') as s:
        s.send_message(msg)

def main():
    critical_devices = [
        '10.0.0.1',  # Core Switch 1
        '10.0.0.2',  # Core Switch 2
        '10.0.1.1',  # Distribution 1
        '10.0.1.2',  # Distribution 2
    ]
    
    failed_devices = []
    for ip in critical_devices:
        if not check_device_health(ip):
            failed_devices.append(ip)
    
    if failed_devices:
        alert_msg = f"CRITICAL: Devices down: {', '.join(failed_devices)}"
        send_alert(alert_msg)

if __name__ == "__main__":
    main()
```
